Basic Calculus and Differentiation Rules

Here’s a question for you: How does a computer learn to solve a problem? For many, a computer learning to solve a problem shows an image of a human-like robot in your mind. Using this analogy, a computer learning to solve a problem would learn exactly like a human, as in it would try to solve the problem based on past experience, figure out what worked well and what did not work well, and adjust it’s approach. As it turns out, this is exactly how machine learning works at a high level.

Now we have to ask ourselves this question: “How can we teach a computer to solve problems like a human?” As it turns out, one of the most efficient ways to do this is to teach the computer to split data. I know that this doesn’t sound related to artificial intelligence, but think of it like this. If the computer can split data, it can tell vanilla ice cream from strawberry ice cream. Let’s say that there is data available on the flavors of ice cream you have purchased in the past, as well as whether the ice cream was bought on a sunny day or a rainy day. Let’s say that we split the data by flavors of ice cream. If we find that most of the vanilla ice cream was bought on rainy days, and most of the strawberry ice cream was bought on sunny days, the computer will find this pattern and will recommend vanilla ice cream to you on rainy days and strawberry ice cream to you on sunny days.

A helpful way to split the data is to plot it on a graph, and to use a line to split the data. Some examples of this are listed below. Click on each screen to see how the data is divided. Take note of what you see after you split the data.

It is helpful for a computer to separate data for us, because computers can analyze thousands to millions of data points very quickly. However, we are now faced with another problem: computers can’t just look at data the way that we might. Computers do not have eyes like we do, therefore they cannot “eyeball” the data. Computers must split the data using math.

Here is how we will split the data using math: We will pick a random line at the start, measure how badly it splits the data, move the line such that we decrease the error, and rinse and repeat. In this way, we will eventually have a line that splits the data very well. Click below to show an animation of a line being moved into place.

However, as before, this raises yet another question: How do we know which direction to move our line so that we can split the data best? It seems that we have just pushed our previous problem forward, and we are still dealing with it now. Computers still do not have eyes, so they have no idea which direction they should move the line. Thankfully, we can seek a solution in calculus, a branch of math concerned with functions.

In essence, how badly our line splits the data is represented as a “cost” function. In order to split the data better, we will take small steps to decrease our cost function until we approach a local minimum. These small decreases to our cost function represent slight movements to our line. Eventually, when our cost function reaches a local minimum, we will have a line that splits the data “well enough”. To decrease our cost function, we will use its derivative.

“Derivative” is a very complex sounding word, but really it’s just the slope of a line at a point. This is very useful to us, because the derivative (slope) of a line at a point always gives the direction of steepest ascent of the function at that point. This means that if the value of the derivative is negative, we should move in the negative x-direction along the function, which in this case, is left. Likewise, if the value of the derivative is positive, we should move in the positive x-direction along the function, which in this case, is right. We move along the x-axis because the x-axis represents the input to the function. Therefore, we can simply subtract the derivative from the current input to the function to move in the direction of steepest descent. I know that this can be a very confusing concept at first, so I have included some examples of different functions below to demonstrate this concept.

Before we begin learning the very surface-level overview of derivatives necessary to learn about neural networks, it’s helpful to learn some vocabulary surrounding derivatives. When we find the derivative of a function, we say that we have “differentiated” it. Derivatives can also be referred to as “tangent lines”, where the tangent line at a point on a function gives the slope of the function at that point. A “secant line” is a line that gives the slope between two points.

Also, I would love to give a huge shout-out to Khan Academy for their clear and interesting explanation of calculus. If you would like to learn more about calculus or get practice at any time, I would encourage you to check out their content at khanacademy.org. For the purposes of this tutorial, however, we will just be covering a very surface level view of differential calculus (the calculus of derivatives).

Additionally, if you would like to learn more about the theory behind calculus, I would highly recommend the YouTube channel 3Blue1Brown, as he explains the theory behind calculus in a fun and interesting way.

First, we will learn that derivative of any number (without a variable) is zero.

For example ...
    The derivative of `0` is `0`.
    The derivative of `1` is `0`.
    The derivative of `2` is `0`.
    The derivative of infinity is `0`.
    The derivative of `-1` is `0`.
    The derivative of `-2` is `0`.
    The derivative of -infinity is `0`.
    and so on and so forth...

Next, we will learn about the power rule. The power rule of differentiation states that the derivative of `x^n` is equal to `n * x^(n-1)`.

For example ...
    The derivative of `x^2` is `2x`.
    The derivative of `x^3` is `3x^2`.
    The derivative of `x^3 + x^2` is `3x^2 + 2x`.
    The derivative of `4x^4` is `4 * 4x^3 = 16x^3`.
    The derivative of `5x^6 + 12x^5` is `30x^5 + 60x^4`.
    The derivative of `x`, which is equal to `x^1`, is `1 * x^0` which is equal to `1`.

What is the derivative of ...
    `3x^3`?
    `10x^2`?
    `20x^4`?
    `x^8`?
    `e*x^6`?

The next topic of interest is the product rule. The product rule of differentiation states that the derivative of `f(x) * g(x)` is equal to `f’(x) * g(x) + f(x) * g’(x)`.

For example ...
    The derivative of `x * x`, which is equal to `x^2`, is `1 * x + x * 1` which is equal to `2x`, reafirming our previous answer.
    The derivative of `x^2 * x`, which is equal to `x^3`, is `2x * x + x^2 * 1` which is equal to `2x^2 + x^2` which is equal to `3x^2`, reafirming our previous answer.

What is the derivative of ...
    `x^2 * x^2`?
    `10x * 20x`?
    `x * 3x^3`?
    `x * x * x * x`?
    `80 * x * sqrt(x)`?

Our next differentiation law is the quotient rule, which states that the derivative of `(f(x) / g(x))` is equal to `(f’(x) * g(x) - f(x) * g’(x)) / (g(x))^2`

For example ...
    The derivative of `x / x`, which is equal to `1`, is `(1 * x - x * 1) / x^2` which is equal to `0 / x^2` which is equal to `0`.
    The derivative of `x^2 / x`, which is equal to `x`, is `(2x * x - x^2 * 1) / x^2` which is equal to `(2x^2 - x^2) / x^2` which is equal to x^2 / x^2 which is equal to `1`.

What is the derivative of ...
    `x^3 / x`?
    `1 / x`?
    `1 / sqrt(x)`?
    `sqrt(x) / x`?
    `20x^4 / 30x^5`?

The final differentiation law we will be covering is the chain rule, which states that the derivative of `f(g(x))` is equal to `f’(g(x)) * g’(x)`.

For example ...
    `f(g(x))` where `f(x) = x^2` and `g(x) = x^2` is equal to `x^4`, and it's derivative is equal to `2(x^2) * 2x` which is equal to `4x^3`, affirming that the derivative of `x^4` is `4x^3`.
    `f(g(x))` where `f(x) = x^2` and `g(x) = sqrt(x)` is equal to `x`, and it's derivative is equal to `2(sqrt(x)) * 1 / (2 * sqrt(x))` which is equal to `1`, affirming that the derivative of `x` is `1`.

What is the derivative of ...
    `f(g(x))` where `f(x) = x^2` and `g(x) = x^3`?
    `f(g(x))` where `f(x) = x` and `g(x) = sqrt(x)`?
    `f(g(x))` where `f(x) = 2x` and `g(x) = 3x`?
    `f(g(x))` where `f(x) = x^(1/3)` and `g(x) = x^(1/4)`?
    `f(g(x))` where `f(x) = 1/x` and `g(x) = 1/x`?

It will also be helpful to memorize that the derivative of `e^x` is equal to `e^x`. For more intuition on why this is true and more information on the mathematical constant `e`, you can learn about calculus on Khan Academy.

Neural Network Theory - How Neural Networks Are Structured

Neural Networks are composed of neurons, which are decimal values from 0 to 1. These neurons are organized in layers. It’s best to think of a layer as a list of neurons. Here is a (simplified) image which will demonstrate this concept visually.

Each neuron’s value is not calculated randomly. The first layer, known as the input layer, has values fed into it. For example, if the neural network was aiming to classify images as either animals or plants, the input layer might contain the grayscale values of every picture in the input image. After the first layer, the other layers are calculated using a weighted sum of the values from the previous layer, plus a number called a bias.

What is a weighted sum, you might ask? To calculate the weighted sum, we will take each neuron from the previous layer, and we will multiply that neuron by a weight, a decimal number from 0 to 1. Then, we will add that multiplied value to a total. When we have done this for every neuron in the previous layer, we will divide the total by the number of neurons in the previous layer. Finally, we will add a bias to the total, which is merely a decimal number from 0 to 1. An interactive dataset with a weighted sum has been included below. Try changing the values of the data, the weights, and the bias to see what happens to the weighted sum!

As you can see, each layer builds on the previous layer using weighted sums. When we get to the output layer, we will make a guess by finding the maximum value from the output layer. For example, in our animal and plant example, there might be two neurons in the output layer so that a high value for the first neuron indicates that the neural network thinks that the image is an animal, and a high value for the second neuron indicates that the neural network thinks that the image is a plant.

One handy aspect of this method is that once we select the neuron in the output layer with the highest value, that neurons value gives us a confidence percentage for our guess. For example, if the highest value in the output layer was 0.94, the neural network is 94% confident that it’s guess is correct. Going back to the animal and plant example, if the first neuron in the output layer has a value of 0.97, that means that the neural network thinks that the image is an animal with 97% confidence.

The cost function tells us how far the neural network is from the correct answer. The cost function is defined by the sum of the squares of the differences between the value of each neuron in the output layer and what that value should be. I know that the previous sentence was very dense and confusing, so I will provide a concrete example to more clearly explain the cost function.

Recall from our animal and plant example that a high value for the first neuron in the output layer meant that the neural network thought that the image was an animal, and that a high value for the second neuron in the output layer meant that the neural network thought that the image was a plant. Let’s say that we fed the neural network an image of an animal. Under these conditions, the value of the first neuron in the output layer should be 1, and the value of the second neuron in the output layer should be 0. Let’s say that the first neuron in the output layer had a value of 0.83 and the second neuron in the output layer had a value of 0.19. The value that the cost function outputs should be `(0.83 - 1)^2 + (0.19 - 0)^2`. If instead the first neuron in the output layer had a value of 0.99 and the second neuron in the output layer had a value of 0.05, the value that the cost function outputs should be `(0.99 - 1)^2 + (0.05 - 0)^2`.

We will initialize all of the weights and biases to random numbers, meaning that our network will perform very poorly. We will use the technique mentioned in the first chapter to minimize the cost function. We will compute the derivatives of the cost function with respect to every weight and bias in the entire neural network. These derivatives are known as partial derivatives, and we will treat every other weight and bias as a constant when calculating them. In other words, we will only apply our differentiation rules for the current weight and bias. For more information on partial derivatives, I would recommend the YouTube channel 3Blue1Brown. We will adjust that weight or bias accordingly by subtracting its derivative from it. Before we subtract that weight or bias’s derivative from it, we will first multiply the derivative by the learning rate, a variable that controls how fast the neural network learns. The learning rate is usually small, so that we take small steps when minimizing the cost function so that we do not minimize the cost function too much and overshoot the local minimum.

Implementing your First Neural Network

Now it’s time for you to implement your first neural network! You will create a neural network that can recognize handwritten digits. This is a common first program in the world of neural networks, as there is a dataset called the MNIST dataset that has already been made out of thousands of images of handwritten digits and their labels (which digit each image represents). For the purposes of this tutorial, I will be using a program that I built a couple of months ago using a Digital Ocean tutorial, with some modifications and explanations of each piece of code.

First you will need to install Python, the programming language that we will use to build our neural network. Python can be installed from it’s website, https:\\www.python.org. Hover over the “Downloads” tab and click on the download for the latest version of Python (Python 3.8.0, as of November 2019). Once the file has been downloaded, double click on it and follow the directions in the wizard to install Python.

Next, you will need to install Tensorflow, the Python library which we will use to program our neural networks. To install Tensorflow, you will want to navigate to their downloads page (https:\\www.tensorflow.org\install). Follow the directions on their website and type in the commands into either a terminal if you are on macOS / Linux or a command prompt if you are on Windows OS.

Now that you have both Python and Tensorflow, it’s time to start writing some code!

First, we have to import the Tensorflow module. Normally, we would use this line:

import tensorflow as tf

However, at this time I am writing this, there is actually a weird bug in tensorflow which causes trouble when using the MNIST dataset. To fix the issue, we must downgrade to tensorflow version 1 using these lines:

import tensorflow.compat.v1 as tf
tf.disable_v2_behavior()

Next, we need to get the MNIST dataset. Because this was such a pain to do, I have provided the tensorflow input_data.py file for you. We can use this file using the following lines:

import input_data
mnist = input_data.read_data_sets('MNIST_data/', one_hot=True)

As mentioned in our theory of neural networks section, we will divide our data into training data (used to train the neural network) and testing data (used to test the neural network’s performance). We will also have a third category called validation data. Validation data is used by tensorflow to figure out how well the network is performing while it is still being trained. The following lines define variables for the number of example data being used for each category.

n_train = mnist.train.num_examples
n_validation = mnist.validation.num_examples
n_test = mnist.test.num_examples

Next, we will need to define the structure of our neural network. Our neural network will have an input layer, 3 hidden layers, and an output layer. The input layer will have enough neurons to store a 28x28 image (784 neurons), the 3 hidden layers will each (somewhat arbitrarily) have 512 neurons, and the output layer will have 10 neurons (to represent which digit the network thinks the input data is, 0 - 9).

n_input = 28 * 28
n_hidden_1 = 512
n_hidden_2 = 512
n_hidden_3 = 512
n_output = 10

The learning rate of our neural network (covered in the chapter on neural network theory) will be very small so that our network trains slowly and confidently.

learning_rate = 0.0001

We will be training our network in “epochs”, which are large sets of images. We will have 10,000 epochs, each with 128 images.

n_epochs = 10000
batch_size = 128

Next we will define the “dropout rate” of our neural network. The dropout rate is a measure of how often a neural network will ignore a weight or bias during training to avoid overtraining that weight or bias (to avoid overfitting the neural network to the training data and to make sure that it can generalize to new data). We will initialize this value to 0.5 to allow this to happen half of the time.

dropout = 0.5

Now we will define placeholder variables that will eventually hold our images, labels, and probability of keeping a specific weight or bias.

images = tf.placeholder('float', [None, n_input])
labels = tf.placeholder('float', [None, n_output])
keep_prob = tf.placeholder(tf.float32)

Next, we will define two dictionaries, for weights and biases respectively. The weights dictionary will contain matrices, or grids, of decimal values, each from -2.0 to 2.0. The biases dictionary will contain lists populated by the value 0.1

weights = {
  'w1':tf.Variable(tf.truncated_normal([n_input, n_hidden_1], stddev=0.1)),
  'w2':tf.Variable(tf.truncated_normal([n_hidden_1, n_hidden_2], stddev=0.1)),
  'w3':tf.Variable(tf.truncated_normal([n_hidden_2, n_hidden_3], stddev=0.1)),
  'out':tf.Variable(tf.truncated_normal([n_hidden_3, n_output], stddev=0.1))
}

biases = {
  'b1':tf.Variable(tf.constant(0.1, shape=[n_hidden_1])),
  'b2':tf.Variable(tf.constant(0.1, shape=[n_hidden_2])),
  'b3':tf.Variable(tf.constant(0.1, shape=[n_hidden_3])),
  'out':tf.Variable(tf.constant(0.1, shape=[n_output]))
}

Now we will calculate each layer using matrix multiplications and additions. Matrix multiplication (for our purposes) is the process of calculating multiple weighted sums of the first matrix, with each row of the second matrix acting as the weights for that sum. In this way, matrix multiplication results in a matrix with a width of one element, where each element is a weighted sum. Matrix addition (for our purposes) is the process of adding each element in one matrix with the corresponding element in the other matrix. If you would like to learn more about matrices and their operations, you can learn about them on Khan Academy.

layer_1 = tf.add(tf.matmul(images, weights['w1']), biases['b1'])
layer_2 = tf.add(tf.matmul(layer_1, weights['w2']), biases['b2'])
layer_3 = tf.add(tf.matmul(layer_2, weights['w3']), biases['b3'])
layer_drop = tf.nn.dropout(layer_3, keep_prob)
output_layer = tf.matmul(layer_3, weights['out']) + biases['out']

Notice that using matrix multiplications and additions, we are able to accomplish the structure discussed in the chapter about neural network theory. The activation of each neuron in layer 1 is calculated to be a weighted sum of the values of the neurons in the input layer (images), plus biases. This is true for all of the layers.

We will define an optimizer using an algorithm called softmax cross entropy with logits, and the optimizer we will use will be the Adam optimizer. For more information on softmax cross entropy with logits and the Adam optimizer, be sure to check out these links:

https://deepnotes.io/softmax-crossentropy https://peterroelants.github.io/posts/cross-entropy-softmax/ https://machinelearningmastery.com/adam-optimization-algorithm-for-deep-learning/ https://towardsdatascience.com/adam-latest-trends-in-deep-learning-optimization-6be9a291375c
cross_entropy = tf.reduce_mean(tf.nn.softmax_cross_entropy_with_logits(labels=labels, logits=output_layer))
train_step = tf.train.AdamOptimizer(learning_rate).minimize(cross_entropy)

correct_pred = tf.equal(tf.argmax(output_layer, 1), tf.argmax(labels, 1))
accuracy = tf.reduce_mean(tf.cast(correct_pred, tf.float32))

Finally, we will run our program using the following lines of code. We will create a session variable and run it to begin training our network. For each datum, we will print out the current iteration number, our network’s loss (the weights and biases that were dropped out), and our network’s accuracy on the training images. After training is completed, we will test the network on all of the test images, printing out the average accuracy at the end.

init = tf.global_variables_initializer()
sess = tf.Session()
sess.run(init)

# train on mini batches
for i in range(n_epochs):
  batch_x, batch_y = mnist.train.next_batch(batch_size)
  sess.run(train_step,
    feed_dict={
      images: batch_x,
      labels: batch_y,
      keep_prob: 1.0
    }
  )

  # print loss and accuracy (per minibatch)
  if i % 100 == 0:
    minibatch_loss, minibatch_accuracy = sess.run(
      [cross_entropy, accuracy],
      feed_dict={
        images: batch_x,
        labels: batch_y,
        keep_prob: 1.0
      }
    )
    print(
      'Iteration',
      str(i),
      '\t| Loss =',
      str(minibatch_loss),
      '\t| Accuracy =',
      str(minibatch_accuracy)
      )

# test the neural network and print it's accuracy.
test_accuracy = sess.run(accuracy, feed_dict={images: mnist.test.images, labels: mnist.test.labels, keep_prob: 1.0})
print('\nAccuracy on test set:', test_accuracy)

Ethics

Here’s an interesting question: All of this stuff about using neural networks to solve problems is all well and good, but why should I care about solving any problems? What if I just want to make the world a better place for myself? I work plenty hard and I deserve it.

Making the lives of other people better feels good. It shows the results of something that you have worked hard on, and creates more joy in a world that certainly needs some. However, even disregarding the moral incentive to use neural networks for good, there is a genuine economic incentive.

As is beautifully explained in this video by Kurzgesagt, economics are fundamentally driven by supply and demand. Therefore, if there is a large demand for a product, a lot of resources will be invested in making it. Because of this principle, it is in your best interest for there to be prosperity all around the world, because if everyone is on a level playing field socioeconomically, there will be incredible amounts of investment in products and solutions that can benefit everyone, which people from all around the world will contribute to, so that those products and solutions are developed faster.

Welcome to the Linux Intel x86-64 Assembly Language Tutorial!

In this tutorial, you will learn the basics of Ubuntu Intel 64-Bit Assembly Language, but also through learning the syntax of this specific program language, you will learn enough to start programming in other assembly languages, provided that you learn the syntax for assembly language on those systems.

What is Assembly Language?

Assembly language is a programming language that is about one step above machine code, the binary code that is executed by your computer directly. This means that assembly language is a fairly low-level programming language. Essentially, in assembly language, you do all memory-management yourself, meaning that you reserve space for variables in your program using a block of data that represents memory on your computer.

Registers

Registers are a way of representing small amounts of data. You access registers via their names, and each register can carry either 8, 16, 32, or 64 bits of data. The University of Las Vegas at Nevada provides a good table for understanding the sizes and names of x86-64 registers in assembly language, which I have included below. You can learn more about assembly language and registers via their PDF linked here.


Parts of an assembly language program

There are three sections to an assembly language program:

The .text section represents what we would traditionally think of as a program, a series of instructions that handle moving data around, conditionals, jumping around in the program, etc.

The .data section represents static data declared in the program. You can think of these data as constants.

The .bss section represents dynamic data reserved by the program. These data are variables.

Assembly Language syntax

Every statement in assembly language (.text section) has the following form:

OPCODE OPERANDS

“Opcode” stands for operator code. Opcodes are instructions which tell the assembler, the compiler (translator) for assembly language, what the program intends to do in this statement. The datum/data that the opcode acts on is/are the operand(s). Here is an example of an assembly language statement:

mov eax, 1

This statement “moves” the value of 1 into the eax register.

Here are some examples of other common assembly language opcodes and their operands.

add <dest>, <src>

The add instruction adds the value in src to the value in dest, and the result is in dest.

sub <dest>, <src>

The sub instruction subtracts the value in src from the value in dest, and the result is in dest.

imul <dest>, <src>

The imul instruction stands for integer multiplication. It multiplies the integer in dest with the integer in src, and the result is in dest.

idiv <dest>, <src>

The idiv instruction stands for integer division. It divides the integer in dest from the integer in src, and the result is in dest.

neg <src>

The neg instruction negates the number in src, and the result is in src. The negation is done using a two’s complement, which you can learn more about on Wikipedia.

call <label>

The call argument goes to the label “label”, keeping track of where “label” was called from, so that a return will return to the original position in the program.

ret

Speaking of which, the ret instruction stands for return, and when called, it will return from the label that was called using call.

push <data>

The space in the registers is limited, too limited for an assembly language program to do very much, which is where the stack comes in handy. The stack is a block of data, and push is an instruction that will add data to the stack, and move the stack pointer (esp), used to keep track of where the next write should be.

pop <register>

The stack works like a stack of papers, meaning that you can also take data from it. This is where the pop instruction comes in handy, as it will move the stack pointer (esp) in the other direction, effectively allowing the data to be overwritten with another push, and placing the now ignored data in the indicated register.

cmp <value1>, <value2>

The cmp instruction compares the two provided values (values are provided either using registers, a location on the stack, or actual raw data). The result of the comparison is stored in a variety of usually unused registers.

jmp <label>

The jmp instruction is like a call that cannot be returned from. It simply “jumps” to a label, which is a named point in the code.

jl <label>

The jl instruction is like the jmp instruction, except for that it also looks at the results of the most recent cmp, and will only execute the jump if value1 was less than value2.

jle <label>

The jle instruction is like the jmp instruction, except for that it also looks at the results of the most recent cmp, and will only execute the jump if value1 was less than or equal to value2.

jg <label>

The jg instruction is like the jmp instruction, except for that it also looks at the results of the most recent cmp, and will only execute the jump if value1 was greater than value2.

jge <label>

The jge instruction is like the jmp instruction, except for that it also looks at the results of the most recent cmp, and will only execute the jump if value1 was greater than or equal to value2.

je <label>

The je instruction is like the jmp instruction, except for that it also looks at the results of the most recent cmp, and will only execute the jump if value1 was equal to value2.

jne <label>

The jne instruction is like the jmp instruction, except for that it also looks at the results of the most recent cmp, and will only execute the jump if value1 was not equal to value2.

Sample Assembly Language Program

Here is a hello world program written in x86-64 assembly language on an Ubuntu computer running an Intel processor. Take note of what you see here that mentioned earlier (opcodes, sections, etc.):

section .text
global _start

_start:
  mov edx, helloworld_length
  mov ecx, helloworld
  mov ebx, 1
  mov eax, 4
  int 0x80

  mov eax, 1
  int 0x80

section .data
helloworld: db 'Hello, world!',10
helloworld_length: equ $ - helloworld

Now, we will walk through the program together. We start out by declaring that we are in the .text section, and then we write “global _start”. This line tells the assembler that we are going to define the starting point for our program later, and is something that must be done at the start of all assembly language programs. Then, we will define a label called _start, by typing “_start:”, with a colon at the end to tell the assembler that this is a label. It is common convention to indent all lines that are contained within the label after that. We will move the length of our string, helloworld_length into edx. We will move the address in memory (the number that indicates where a piece of data is stored on our computer) of our string helloworld into ecx. We will move 1 in into ebx to let the assembler know that we want to write to standard output. We will move 4 into eax to let the assembler know that the operation that we want to perform is a write. Finally, we will call int 0x80. “int” stands for interrupt, and the argument 0x80 is used for system calls. When we execute a system call, the computer will look at the values inside of the registers to determine what should be done. We have gone through those values already. Next, we will quit the program, so we will move 1 into eax to specific an exit, and call int 0x80 again. The final part of our program is the .data section, where we define our string using db (which defines one or more bytes of data), and the length of our string using equ (which is the equivalent of a normal equals sign). Note that the reason that we have 10 after our string is because 10 represents a newline. Also note that the way that we find the length of the string is $ - the name of the string. This will only work on the line right after the string is defined, because $ means the current location, so to find the length of the string, we will subtract the current location in memory from the location in memory of the start of the string.
